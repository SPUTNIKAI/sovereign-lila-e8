# Contributing to Sovereign-Lila-E8

Thank you for your interest in contributing to the Sovereign-Lila-E8 project! We welcome contributions of all kinds: bug reports, feature suggestions, documentation improvements, and code changes.

Here’s how you can help:

## 🐛 Reporting Bugs

1. **Check** if the bug has already been reported in the [Issues](https://github.com/SPUTNIKAI/sovereign-lila-e8/issues).
2. If not, open a new issue using the provided template.
3. Please include:
   - A clear description of the bug.
   - Steps to reproduce the behavior.
   - What you expected to happen instead.
   - Versions of relevant libraries (Python, PyTorch, etc.).

## 💡 Suggesting Enhancements

1. First, check if the enhancement has already been suggested in the Issues.
2. Describe the problem you are trying to solve and your proposed solution.
3. If possible, provide code examples or mockups to illustrate the idea.

## 🔧 Pull Requests

1. **Fork** the repository and clone it locally.
2. Create a new branch for your changes (e.g., `feature/amazing-idea` or `fix/bug-name`).
3. If your change adds new functionality, include tests that cover it.
4. Update the documentation (README, docstrings, etc.) to reflect your changes.
5. Run the test suite to ensure everything passes:
   ```bash
   python -m pytest tests/
Open a pull request against the main branch with a clear description of your changes.

## 🧪 Testing
All new features and bug fixes must include appropriate tests.
The project uses pytest; you can run the full test suite locally with the command above.
Make sure your code adheres to the existing style (PEP 8 is a good baseline).

## 📝 Commit Messages
Write clear, concise commit messages.

Use the present tense ("Add feature" not "Added feature").

Reference issue numbers when applicable (e.g., "Fixes #123").

## 📝 Contributor License Agreement

Before we can accept your pull request, you need to sign our Contributor License Agreement (CLA). 
This ensures that we have the necessary rights to use your contribution under both our open-source 
(AGPLv3) and commercial licenses.

### How to sign:
1. When you submit your first pull request, our CLA bot will automatically ask you to sign.
2. Click the link, review the agreement (it takes ~2 minutes), and sign.
3. That's it! Your PR will then be reviewed.

### Why do we need this?
Sovereign-Lila-E8 uses a dual-license model (AGPL for open source + commercial for enterprises). 
The CLA allows us to include community contributions in both versions, ensuring the project can 
be sustainable while remaining open. This is the same model used by Qt, MySQL, and MongoDB.
